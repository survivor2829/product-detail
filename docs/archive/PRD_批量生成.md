# 批量导入一键生图功能 需求文档

> 生成日期：2026-04-19  
> 状态：初版，待用户确认后进入开发

---

## 一句话定义

批量上传多个产品文件夹 → 先全部出HTML版 → 用户筛选后AI精修 → 打包下载。

---

## 目标用户

小玺AI内部同事，需要一次性为大量产品生成详情页的场景：
- 新品上线季，需要给20-50个产品同时出图
- 老产品批量升级视觉风格
- 客户委托一次性交付多个产品的详情页

---

## 核心功能（必须实现）

### F1. 文件夹上传
- 前端"选择文件夹"按钮，点击后调用浏览器File API的 `webkitdirectory` 属性
- 浏览器自动遍历文件夹内所有文件，打包上传到服务器
- 支持进度显示（上传中.../解析中.../待处理）
- 上传完成后显示识别到的产品数量和产品列表

### F2. 文件夹结构约束（统一命名规则）
每个产品一个子文件夹，内部命名规则：
```
批量任务/
├── 产品A/
│   ├── product.jpg     # 必须：主图，支持jpg/png/webp
│   ├── product_2.jpg   # 可选：细节图2（specs屏用）
│   ├── product_3.jpg   # 可选：细节图3
│   └── desc.txt        # 必须：产品文案，UTF-8编码
├── 产品B/
│   ├── product.jpg
│   └── desc.txt
└── ...
```
- 文件夹名作为产品的显示名称
- **product.jpg是主图**，用于hero屏和cta屏
- **product_2.jpg、product_3.jpg...是细节图**，用于specs屏和场景屏展示产品的不同角度/功能部位
- 主图和文案文件名固定，没有主图或文案的产品自动跳过并在UI上标注"命名不规范"
- 下载模板压缩包给同事参考：`batch_template.zip`

### F3. 批量数量限制
- 单次批量**最多50个产品**
- 超过50个系统拒绝上传，提示用户分批传
- UI上实时显示"已识别 X/50 个产品"

### F4. 任务队列 + 小并发处理
- **独立资源池**：批量任务和单品任务使用各自独立的队列，互不阻塞。用户在跑50产品批次时，另一个同事仍然可以正常用单品功能
- 使用 Python `threading.ThreadPoolExecutor` + 内存队列实现（轻量级，不用装Redis）
- 并发数 = 3（可配置）
- 每个产品的处理流程：
  ```
  读取产品图和文案
    ↓
  DeepSeek解析文案 → 生成parsed_data
    ↓
  rembg抠图 → 生成透明底产品图
    ↓
  渲染HTML模板 → Playwright截图 → 拼接长图
    ↓
  保存到 batch_{batch_id}/product_{N}/html.jpg
    ↓
  更新任务状态为"HTML完成"
  ```
- HTML版完成后产品进入"待AI精修"状态，等待用户手动触发

### F5. 实时预览界面
批次运行时，网页上实时展示：
```
批次 #batch_20260419_001  进度: 15/50
├── ✅ 产品A - HTML完成 [预览] [勾选AI精修]
├── ⏳ 产品B - 处理中...
├── ✅ 产品C - HTML完成 [预览] [勾选AI精修]
├── ❌ 产品D - 失败（命名不规范，跳过）
└── ...
```
- 每个产品完成后立即显示缩略图
- 用户可以随时勾选/取消勾选"需要AI精修"
- 支持"全选待AI精修"和"取消全选"
- **实时通信：使用 WebSocket** 推送进度更新到前端（低延迟，不浪费流量）
- **浏览器关闭后任务暂停：** 如果用户关闭了浏览器（WebSocket断开），服务器会暂停该批次的处理；用户重新打开页面进入批次页后，WebSocket重连，任务自动继续

### F6. AI精修批量触发
- 用户勾选完需要AI精修的产品后，点击"开始AI精修"按钮
- **费用预估与确认弹窗：** 点击后弹出确认框，显示：
  ```
  即将对 X 个产品进行AI精修
  预估消耗：X × 6 = XX 次豆包Seedream API调用
  预估费用：约 ¥XX.XX 元
  预估耗时：约 X 分钟
  [取消]  [确认开始]
  ```
  （API单价按豆包实际计费规则动态计算）
- 用户确认后系统把勾选的产品加入AI精修队列，3并发处理
- AI精修流程：调用豆包Seedream生成7屏背景 + HTML/CSS合成 + Playwright截图
- 每个产品AI精修完成后在UI上标注"AI精修完成"

### F7. 失败容错
- 单个产品失败**不阻塞**整个批次
- 失败类型分类记录：
  - 命名不规范（跳过前就能识别）
  - DeepSeek解析失败
  - rembg抠图失败
  - API超时
  - 未知错误
- 最终生成《失败报告》供用户查看和重试失败的单个产品

### F8. 打包下载
- 批次全部完成后，用户可以：
  - 下载全部HTML版（zip）
  - 下载全部AI精修版（zip）
  - 下载混合版（每个产品只保留用户选的那个版本）
- zip内部目录结构：
  ```
  batch_20260419_001/
  ├── 产品A/
  │   ├── html.jpg
  │   └── ai_refined.jpg  # 如果勾选了AI精修
  ├── 产品B/
  │   └── html.jpg
  └── 失败报告.txt
  ```

### F9. 历史批次管理
- 新增"历史批次"页面，列出所有已完成的批次
- 每条记录显示：批次ID、批次名称、创建时间、产品数量、状态、操作按钮
- 支持：查看详情、重新下载、删除批次
- 文件保留期：30天（超过自动清理）

### F10. 批次命名规则
- **用户自定义批次名称**：上传文件夹后，弹出输入框让用户给批次起名字（例如："Q2新品批次"、"阳光集团项目"）
- 默认值：上传的根文件夹名称
- 同一用户同一天允许创建多个批次，不做限制
- **同一天多批次的显示方式：**
  ```
  [2026-04-19] Q2新品批次          45个产品  已完成
  [2026-04-19] Q2新品批次 (第2次)   12个产品  运行中
  [2026-04-19] 临时测试            5个产品   已完成
  [2026-04-18] 阳光集团项目         30个产品  已完成
  ```
  系统自动给同名批次后面加 `(第N次)` 后缀区分

### F11. 模板选择策略
- **AI自动匹配模板**：上传时默认"智能匹配"模式，系统根据DeepSeek解析出的产品类型自动选择合适的模板（例如：清洁设备→经典红、智能机器人→科技蓝、高端工具→黑金）
- **用户可覆盖**：如果用户想统一风格，上传界面提供可选的"统一模板"下拉框，选了之后整个批次都用这个模板
- **自动匹配规则表**（后续迭代中可以调整）：
  ```
  清洁设备类 → classic-red
  智能机器人/AI设备 → tech-blue
  高端商用工具 → premium-gold
  环保/绿色产品 → fresh-green
  工业用品 → industrial-gray
  默认 fallback → classic-red
  ```

## 明确排除（不做的事）

1. **单批超过50个** — 服务器内存限制，超过会崩
2. **Excel/CSV批量导入（本期）** — 文件夹方式统一，避免多种入口维护成本高
3. **大并发（10+）** — 服务器2GB内存扛不住
4. **全量AI精修** — 用户必须手动筛选，控制API成本
5. **暂停/继续功能（本期）** — 实现复杂，浏览器关闭=自动暂停，重开=自动继续已够用
6. **手动字段映射** — 文案格式统一由DeepSeek自动解析
7. **文件大小限制** — 不做上限限制，服务器能处理多大就支持多大
8. **文案长度限制** — 不做限制，DeepSeek能处理多长就支持多长
9. **特殊字符处理（本期）** — 默认DeepSeek能正确处理表情符号、HTML标签等，如实际出问题再做

## 技术可行性备注

- [验证日期：2026-04-19] 条件：Chrome 120+、Edge 120+
  - webkitdirectory API能识别文件夹结构 → 成立
  - Safari支持但有兼容问题 → 建议提示用户用Chrome/Edge
  
- [验证日期：2026-04-19] 条件：腾讯云服务器2GB内存
  - 3并发处理 × 每个产品约500MB峰值 → 约1.5GB峰值 → 成立
  - 5并发 × 500MB = 2.5GB → 超内存风险 → 暂用3并发保守起见
  
- [验证日期：2026-04-19] 条件：现有Flask架构
  - Flask原生不支持长任务 → 需引入Celery+Redis 或 RQ 或 线程池
  - 推荐用 **threading.ThreadPoolExecutor + 内存队列** 最轻量，不用装Redis
  
- [待验证] 条件：Playwright Chromium实例跨产品复用
  - 单实例跨产品截图能否稳定 → 需要在开发阶段验证
  - 如果不稳定，每个产品独立启动实例（慢但稳）

- [验证日期：2026-04-19] 条件：Flask + WebSocket
  - 需要引入 `flask-socketio` 或 `flask-sock` 库
  - WebSocket断开时触发批次暂停逻辑，重连时恢复
  - 如果客户浏览器代理或防火墙屏蔽WebSocket，降级到轮询兜底（3秒一次）

- [验证日期：2026-04-19] 条件：豆包Seedream API计费
  - 需要在费用预估弹窗中动态显示当前API单价
  - 单价从配置文件读取，定期手动更新（豆包价格可能变化）

## 用户流程

### 同事的完整操作路径

```
① 组织文件夹（在本地电脑上）
   创建"批量任务"文件夹
   每个产品一个子文件夹
   - 必须：product.jpg（主图）+ desc.txt（文案）
   - 可选：product_2.jpg、product_3.jpg（细节图）
   最多50个产品

② 上传（在网站上）
   进入"批量生成"页面
   点"选择文件夹"按钮，选中"批量任务"文件夹
   看到识别结果："识别到45个产品，3个命名不规范将跳过"
   给批次起名（默认用根文件夹名）
   选择模板策略（默认：AI自动匹配；可选：统一用某个模板）
   点"开始处理"

③ 等HTML版出（15-30分钟）
   页面通过WebSocket实时显示每个产品的HTML缩略图
   用户边看边勾选哪些值得AI精修
   可以中途关浏览器去做别的，回来还在跑

   特别说明：如果关闭浏览器，服务器自动暂停该批次
   重新打开页面进入批次，WebSocket重连后自动继续处理

④ AI精修（可选，另外20-40分钟）
   勾选完后点"开始AI精修"
   弹出费用预估确认框：
   "即将对 20 个产品进行AI精修
    预估消耗 120 次API调用，约 ¥XX.XX
    预估耗时约 30 分钟"
   确认后开始处理
   网页显示"正在精修: 10/20..."

⑤ 下载
   全部完成后点"打包下载"
   选下载格式（全HTML / 全AI / 混合）
   得到一个zip

⑥ 事后查看
   之后任何时候都可以进"历史批次"页面
   30天内的批次都能找到并重新下载
   同一天多个批次自动加 (第N次) 后缀区分
```

## 开发阶段拆解

### 阶段一：后台基础（最核心）
1. 文件夹上传API + 文件夹结构解析（支持主图+细节图）
2. 双资源池任务队列（批量池 + 单品池，ThreadPoolExecutor实现）
3. 批次状态存储（数据库表 batches 和 batch_items）
4. 单个产品HTML版处理（复用现有能力）

### 阶段二：前端UI
5. "批量生成"页面 + 文件夹上传组件
6. WebSocket通信 + 浏览器连接状态监控
7. 产品列表 + 实时进度 + 勾选AI精修
8. 批次命名交互 + 模板选择交互（AI自动/统一）

### 阶段三：AI精修批量
9. 模板智能匹配逻辑
10. 费用预估弹窗
11. AI精修队列（串联到主流程）
12. 失败跳过和报告逻辑

### 阶段四：交付
13. 打包下载（zip生成，支持混合版）
14. 历史批次页面（同名批次(第N次)后缀）
15. 30天清理定时任务

### 阶段五：打磨
16. 批次模板压缩包下载
17. 失败报告改善
18. WebSocket断线降级轮询兜底
19. 边界情况处理
